# AOP-NP-instances
一、算例生成
鉴于现有标准算例库无法直接适配本文研究的多重图场景，本文基于经典 VRP 基准构建了包含 43 组测试算例的专用数据集，分为小规模与大规模两类，另设 5 组独立算例用于LrMA的个体效能评估。所有算例均遵循“X”命名规范，其中 X 代表节点规模。
data_S：该数据集基于cvrp 基准算例中的Augerat等人（1995）所提出的VRP实例中包含站点数n=32-65个节点的小规模的算例。采取了以下设置来适用于本问题：(a)针对坐标求取欧几里得距离，然后生成距离矩阵，对矩阵施加（-0.1，0.1）的随机扰动构成非对称矩阵；（b）根据距离矩阵生成多重图中两条弧的分数与时间，时间优先弧（时间短、分数低）的分数与时间系数分别是0.1、0.1，分数优先弧的分数（reij1）与时间系数是0.3、0.2；(c)节点分数为随机生成，其中有30%的高分数站点，高分数站点的分数为100，低分数站点的分数为1，另外节点的游览时间统一设为0。
data_L：该数据集基于基于cvrp 基准算例中的Uchoa等人（2017）所提出的VRP实例，生成了包含101-701个节点的23个算例。数据的设计均同data_S。
data_E：该数据基于tsp95中的kroA100、kroB100、kroC100、kroD100、kroE100进行生成。为增大求解难度，防止算法快速收敛，故调整了两条弧的分数与时间的系数，时间优先弧（时间短、分数低）的分数与时间系数分别是0.12、0.32，分数优先弧的分数（reij1）与时间系数是0.35、0.78，并保留4位小数；同时将节点的游览时间调整为节点分数的1.6倍，节点分数依旧随机生成。
二、算例格式
第1行：数据集标题及骑行节点数量（n）说明；

第2至3行：总预算时间阈值（T）标签及其数值；

第4至5行：节点分数；

第6至7行：节点时间；

第8行至第9+n行：时间优先弧的标签与分数矩阵；

第10+n至第11+2n行：分数优先弧的标签与分数矩阵；

第12+2n行至第13+3n行：时间优先弧的标签与时间矩阵；

第14+3n行至第15+4n行：分数优先弧的标签与时间矩阵

原始数据：
data_S 与 data_L 分别见 https://galgos.inf.puc-rio.br/cvrplib/index.php/en/instances 中的 Set A (Augerat et al., 1995) 与Set X (Uchoa et al., 2017)
data_E 见 https://github.com/shredderzwj/TSPLIB?tab=readme-ov-file 中的 kroA100、kroB100、kroC100、kroD100、kroE100
